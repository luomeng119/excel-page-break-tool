# 卷内目录分页系统 - PRD v2

## 1. 项目概述

### 1.1 项目背景
档案管理中，卷内目录Excel文件包含多个案卷，每个案卷的文件从序号1开始编号。打印或归档时，需要在每个新案卷开始处插入分页符，确保每个案卷单独成页。

### 1.2 核心需求（用户原始需求）
> **当 Excel 中的"顺序号"列值变为 001（即1）时，在该行前插入分页符。**

### 1.3 已有数据特征
| 属性 | 值 |
|------|-----|
| 文件 | 卷内目录.xlsx |
| 总行数 | 10,267 行（含2行表头） |
| 案卷数 | **363 卷** |
| 每卷起始行 | 顺序号 = 1 |
| 数据列 | 顺序号、责任者、题名、日期、页号、备注 |

### 2.1 功能列表

|---|------|------|--------|
| F1 | 文件上传 | 拖拽/点击上传 .xlsx/.xls 文件 | P0 |
| F2 | 数据预览 | 表格展示原始数据，支持滚动浏览 | P0 |
| F3 | 分页预览 | 可视化标记所有分页位置（序号=1的行前） | P0 |
| F4 | 卷统计 | 显示总卷数、总文件数、分页点数量 | P0 |
| F5 | 分卷切换 | 点击卷号快速跳转到对应卷 | P1 |

### 2.2 用户交互流程

```
[上传文件] → [解析Excel] → [识别分页点] → [展示预览]
                                                        ↓
                                                   [导出/打印]
```

### 2.3 分页识别规则
- **触发条件**：当前行顺序号 = 1 **且** 上一行顺序号 ≠ 1
- **分页位置**：在该行**上方**插入分页符
- **首行处理**：第1条数据（顺序号=1）不触发分页（文件开头无需分页符）

### 3.2 分页标记数据结构
```typescript
interface Volume {
  volumeNo: number;       // 卷号（从1开始）
  startRow: number;      // 数据行索引（0-based，不含表头）
  endRow: number;        // 结束行索引
  recordCount: number;   // 该卷文件数量
}

interface PageBreak {
  rowIndex: number;       // 要插入分页符的行（0-based）
  volumeNo: number;       // 所属卷号
}
```

- 总卷数：363 卷
- 分页点数量：362 个（每卷开始处，第1卷不计入）

## 5. 技术方案

### 5.1 技术栈
- **纯前端**：单HTML文件，无后端依赖
- **Excel解析**：SheetJS (CDN引入)
- **样式**：内联CSS，简洁美观
- **交互**：原生JavaScript

### 5.2 分页算法
```javascript
function findPageBreaks(data) {
  const breaks = [];
  for (let i = 1; i < data.length; i++) {
    if (data[i].顺序号 === 1 && data[i-1].顺序号 !== 1) {
      breaks.push({ rowIndex: i, volumeNo: breaks.length + 2 });
      // 第1卷不需分页符，从第2卷开始
    }
  }
  return breaks;
}
```

### 5.3 Excel分页符写入
```javascript
// 使用SheetJS的pageBreak属性设置分页符
worksheet['!pageBreak'] = breaks.map(b => b.rowIndex + 1); // 行号+1（Excel 1-based）
```
